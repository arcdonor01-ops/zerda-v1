Phase 1: Reconnaissance

The foundation of every successful bug bounty hunt. Gather as much information as possible about your target.

Subdomain Enumeration

Discover all subdomains associated with your target. Subdomains often host forgotten services, development environments, or less-secured applications.

Tools:
Subfinder Amass Assetfinder Knockpy
Example Commands:
subfinder -d target.com -all
amass enum -d target.com -o subdomains.txt
assetfinder --subs-only target.com

Port Scanning

Identify open ports and running services. Non-standard ports often reveal interesting services like development tools, admin panels, or backup services.

Tools:
Nmap Masscan Naabu Rustscan
Example Commands:
nmap -sV -p- target.com
nmap -sC -sV -p 1-10000 target.com
masscan -p1-65535 target.com --rate=1000

OSINT Gathering

Use open-source intelligence to find information about the target. This includes employee emails, technology stacks, and infrastructure details.

Tools:
Maltego Shodan Censys theHarvester
Example Commands:
theHarvester -d target.com -b all
shodan search org:"Target Corp"
maltego -t target.com

Directory Brute-forcing

Discover hidden directories and files. Many applications have admin panels, backup files, or debug endpoints that aren't publicly linked.

Tools:
Gobuster Dirb Ffuf Dirsearch
Example Commands:
gobuster dir -u target.com -w wordlist.txt
ffuf -w wordlist.txt -u https://target.com/FUZZ
dirb https://target.com/ wordlist.txt

Phase 2: Discovery

Identify live assets, analyze technologies, and map the attack surface. Focus on what can be attacked.

HTTP Probing

Check which subdomains are actually live and responding. Filter out dead hosts to focus your testing on active services.

Tools:
httpx httprobe EyeWitness Aquatone
Example Commands:
httpx -l subdomains.txt -silent
httprobe < subdomains.txt
eyewitness --web -f subdomains.txt

Technology Stack Analysis

Identify the technologies used by the target. Knowing the stack helps you focus on technology-specific vulnerabilities and exploits.

Tools:
Wappalyzer BuiltWith WhatWeb Nuclei
Example Commands:
whatweb target.com
nuclei -l targets.txt -t technologies/
wappalyzer https://target.com

JavaScript Analysis

Analyze JavaScript files for hidden endpoints, API calls, and sensitive information. Modern SPAs often leak valuable information in client-side code.

Tools:
JS-Scan LinkFinder SecretFinder Grep
Example Commands:
grep -r "api/" *.js
python3 linkfinder.py -i https://target.com
secretfinder -i target.com -o cli

API Discovery

Find API endpoints that might not be documented. APIs often have different security controls and can be a rich source of vulnerabilities.

Tools:
Kiterunner Arjun Postman Insomnia
Example Commands:
arjun -u https://target.com/api/
kr scan api.txt -w routes.txt
ffuf -w api-wordlist.txt -u https://target.com/FUZZ

Phase 3: Testing

Systematically test for vulnerabilities. Combine automated scanning with manual testing for best results.

Cross-Site Scripting (XSS)

Test for both reflected and stored XSS vulnerabilities. Modern applications often have complex input validation that can be bypassed.

Tools:
XSStrike XSSer Burp Suite Dalfox
Testing Approach:
1. Test all input fields and parameters
2. Try different XSS payloads
3. Check for WAF bypass techniques
4. Test DOM-based XSS in JavaScript

SQL Injection

Test for SQL injection in all database interactions. Even with modern frameworks, SQL injection remains a common vulnerability.

Tools:
SQLMap Burp Suite NoSQLMap Manual Testing
Testing Approach:
1. Test all parameters and form fields
2. Use error-based detection
3. Try time-based blind SQLi
4. Test different database types

Business Logic Testing

Test the application's business logic for flaws. These vulnerabilities are often overlooked by automated tools and can have high impact.

Tools:
Manual Testing Burp Suite Postman Custom Scripts
Testing Areas:
‚Ä¢ Authentication bypass
‚Ä¢ Privilege escalation
‚Ä¢ Payment flow manipulation
‚Ä¢ Rate limiting bypass

Access Control Testing

Test for IDOR (Insecure Direct Object Reference) and other access control issues. These are common in applications with complex user roles.

Tools:
Burp Suite Custom Scripts AutoRepeater Manual Testing
Testing Approach:
1. Test with different user accounts
2. Modify object IDs in requests
3. Test horizontal/vertical privilege escalation
4. Check for forced browsing

Phase 4: Reporting

A good report can make the difference between a bounty and a rejection. Learn to document effectively.

Report Structure

Follow a clear structure that makes it easy for triagers to understand and reproduce your finding.

Essential Sections:
1. Summary - Brief vulnerability overview
2. Description - Technical details and impact
3. Steps to Reproduce - Clear, numbered steps
4. Proof of Concept - Screenshots, videos, payloads
5. Impact Assessment - Business impact explanation
6. Remediation - Suggested fix (optional)

Proof of Concept

Provide clear evidence that demonstrates the vulnerability. Visual proof is often more convincing than written descriptions alone.

Evidence Types:
‚Ä¢ Screenshots with annotations
‚Ä¢ Screen recordings (GIF/MP4)
‚Ä¢ Burp Suite request/response pairs
‚Ä¢ Custom exploit scripts
‚Ä¢ Log files and error messages

Severity Assessment

Accurately assess the severity of your finding. Understand CVSS scoring and consider the business context of the vulnerability.

Severity Factors:
‚Ä¢ Ease of exploitation
‚Ä¢ Authentication requirements
‚Ä¢ Potential impact (data, systems, users)
‚Ä¢ Business context and asset value
‚Ä¢ WAF/IDS bypass requirements

Communication Best Practices

Maintain professional communication throughout the disclosure process. Good relationships lead to more opportunities and higher bounties.

Do's and Don'ts:
‚úÖ Be patient and professional
‚úÖ Respond promptly to questions
‚úÖ Provide additional information when requested
‚ùå Don't rush triagers or demand updates
‚ùå Don't argue about severity ratings
‚ùå Don't disclose vulnerabilities publicly

Key Success Factors

üîÑ

Consistency

Follow the methodology consistently. Don't skip phases or rush through reconnaissance.

üìù

Documentation

Keep detailed notes throughout the process. Good documentation leads to better reports.

üéØ

Focus

Spend quality time on each target. Deep understanding beats broad but shallow testing.

üìö

Learning

Continuously learn from write-ups, reports, and other hunters. The field evolves rapidly.
